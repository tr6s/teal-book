## この章でできるようになること

- Teal が連携できる5つの連携先（Google・LINE・Slack・Notion・X）とそれぞれの用途がわかる
- アカウント連携の承認手順がわかる
- 「Slack で指示 → Notion に保存 → LINE で通知」のようなクロスプラットフォーム連携を組める

![Tealを中心とした外部連携ハブ](/images/teal-user-guide/09-integrations-hub.png)

## 「実行できる AI」を支える連携

Teal が単なるチャット AI と違うのは、**外部サービスと接続して実際にアクションを取れる** ことです。連携を済ませておくと、AI が「カレンダーに予定を追加」「Notion にタスクを登録」「X に投稿」といった具体的な動作を、ユーザーの代わりに実行します。

連携設定はサイドバーの「外部連携」タブから数クリックで行えます。各サービスの公式ログイン画面で承認する仕組みなので、Teal がパスワードを保持することはありません。

## 5つの連携先

| 連携先 | 接続内容 | 主な用途 |
| --- | --- | --- |
| Google | Google アカウントと連携（カレンダー / Gmail / ドライブ） | 予定の確認・追加、メールの読み取り・ドラフト作成・アドレス抽出、ファイルの参照・保存 |
| LINE | LINE 公式アカウント経由 | モバイルからの対話・通知受信 |
| Slack | Slack API と連携 | メッセージ通知・チャンネル連携・チーム内活用 |
| Notion | Notion API と連携 | タスク管理・データベース操作・ナレッジ管理 |
| X（旧Twitter） | X API と連携 | 自動投稿・エンゲージメント分析・競合監視 |

「会話に使う系（LINE / Slack）」「データを読み書きする系（Google / Notion）」「発信に使う系（X）」と、3つの役割に分けるとイメージしやすいです。Google は内部に Calendar / Gmail / Drive の3つを束ねた集合体として、まとめて1つの連携先として扱います。

## 操作手順

### Step 1. 連携メニューを開く

1. Web ダッシュボードのサイドバーから「外部連携」を開きます
2. 連携可能なサービスがアイコン付きで一覧表示されます
3. 接続したいサービスの「接続」ボタンをタップします

### Step 2. 公式ログイン画面で承認する

1. ボタンを押すと、対象サービス（Google / Notion / Slack など）の **公式ログイン画面** に遷移します
2. ログイン後、Teal がアクセスを求めている権限（カレンダー閲覧・メール送信など）が表示されます
3. 「許可」をタップすると、Teal の画面に戻り「連携済み」と表示されます

各サービスの公式ログイン画面で承認する仕組みなので、ユーザーのパスワードを Teal に渡す必要はありません。権限の取り消しも、各サービス側のアカウント設定からいつでも可能です。

### Step 3. 連携できているか確かめる

連携直後は、AI に簡単な指示を出して動作確認するのがおすすめです。

```text
（Googleカレンダー連携後）
今週の予定を箇条書きで教えて
```

```text
（Notion 連携後）
私の Notion ワークスペースで「タスク」と名前のついた
データベースが見つかるか教えて
```

応答が返ってくれば、連携は成功しています。エラーが返るときは、Step 2 の承認画面で許可した権限が足りない可能性があります。

## クロスプラットフォーム連携の例

複数の連携を組み合わせると、1つのワークフローを横断的に扱えます。

| 起点 | 中継 | 終点 |
| --- | --- | --- |
| Slack で「議事録をまとめて」と指示 | AI が会議の音声 / メモを解釈 | Notion のデータベースにタスク化して保存 |
| LINE で「明日の予定を確認」 | Googleカレンダーから取得 | LINE に箇条書きで返答 |
| 自動タスクで競合の X 投稿を分析 | Gmail からも関連メールを抽出 | Slack チャンネルに日次レポートを投稿 |

「指示の入口（Web ダッシュボード / LINE / Slack）」と「連携先（Google / LINE / Slack / Notion / X の5つ）」の **両方がマルチ対応** なので、起点・中継・終点を自由に組み替えられるのが Teal の特徴です。

## つまずきやすいポイント

:::message
外部サービス連携は、各サービス側の **アカウントの種類** によって挙動が変わります。Google は個人アカウントと Google Workspace（会社）アカウントで権限の範囲が違うので、業務用途では Workspace アカウントで連携することをおすすめします。
:::

:::message alert
LINE と Slack は「指示の入口」と「通知先（連携先）」の両方で登場します。第04章で扱ったのは **AI とユーザーが会話する経路** としての利用、本章で扱うのは **AI から通知やメッセージを送る連携先** としての利用です。同じサービスでも、どちらの文脈で使うかを意識すると整理しやすくなります。
:::

:::details 連携を解除したいとき
1. Teal 側 → サイドバー「外部連携」→ 該当サービスの「切断」をタップ
2. 各サービス側 → アカウント設定の連携アプリ一覧から Teal を取り消し

両方で解除すると、確実にアクセス権を切れます。
:::
