## この章でできるようになること

- 「役割」とは何か、カテゴリとの違いがわかる
- 自分の業務に合わせた役割を作成・編集できる
- 役割を切り替えながら複数の専門分野を使い分けられる

![役割の作成フローと使用例](/images/teal-user-guide/06-roles-flow.png)

## 「役割」とは

**役割** は、AI アシスタントに「専門分野」を持たせる機能です。たとえば「SNS集客サポート」「契約書ドラフト作成」「営業リスト作成」といった専門領域ごとに役割を作っておくと、サイドバーから **ワンタップでその専門の会話を再開** できます。

カテゴリ（第05章）との違いはシンプルです。

- **カテゴリ**：1回の会話の **始め方** を提示する
- **役割**：複数の会話で **同じ専門性を持たせ続ける** ための設定

「いつも同じ業種・同じ立場で AI と話したい」「会話のたびに前提を説明し直すのが面倒」という方は、役割を作っておくと毎回の入力が減ります。

## 操作手順

### Step 1. 役割を新規作成する

1. Web ダッシュボードのサイドバーから「役割」を開きます
2. 右上の「＋ 新しい役割」ボタンをタップします
3. 役割の **名前** と **説明（指示文）** を入力します

役割の指示文は、AI にとっての「常時参照される前提」になります。たとえば次のような書き方をします。

```text
あなたは中小企業のSNS集客を支援する専門家です。
ユーザーは BtoB 製造業の経営者で、X（旧Twitter）と
LinkedIn を主軸に運用しています。投稿文は丁寧体で、
1投稿あたり120〜140字、ハッシュタグは最大3個までで提案してください。
```

具体的な制約（文字数・トーン・対象媒体）まで書くと、AI の応答がぶれにくくなります。

### Step 2. 役割を保存・呼び出す

1. 入力が終わったら「保存」をタップします
2. サイドバーの「役割」一覧に新しい項目が追加されます
3. 一覧から役割をタップすると、その役割が適用された新しい会話が始まります

役割は **ユーザーごと** に保存されます。1つのアカウントで複数の役割を作っておき、目的に応じて切り替える、という使い方が想定されています。

### Step 3. 既存の役割を編集する

役割の指示文は何度でも書き直せます。

1. サイドバーの「役割」一覧から、編集したい役割の右側の三点メニューを開きます
2. 「編集」をタップして指示文を書き直します
3. 「保存」で上書きされ、以後の会話に反映されます

過去の会話には遡って影響しません。編集後の役割は、編集後に始める新しい会話から有効になります。

## 役割の使用例

次のような役割が用意されています。自分の業務に近いものから真似していくと、最初の役割を作りやすいです。

| 役割名 | 機能内容 |
| --- | --- |
| SNS集客サポート | SNS 戦略の立案、投稿文作成、ハッシュタグ提案 |
| X 自動投稿 | X（旧Twitter）への定期的な自動投稿を実行 |
| Notionタスク管理 | Notion のデータベースと連携したタスク管理 |
| 営業リスト作成 | 条件に合う見込み先リストを自動生成 |
| 契約書ドラフト作成 | 契約書のドラフトを自動生成、条項提案 |
| プレゼン資料作成 | スライド構成の提案、原稿作成 |
| 資金調達戦略立案 | 資金調達の戦略立案、投資家向け資料のアドバイス |
| 天気・服装アドバイス | 天気予報に基づく服装の提案 |
| 毎日筋トレの工夫 | 日替わりのトレーニングメニューを提案 |

「X 自動投稿」のように、役割が **自動タスク（第08章）** と紐づくものもあります。役割は「AI の人格」であり、自動タスクは「定期実行のスケジュール」、と分けて考えるとわかりやすいです。

## つまずきやすいポイント

:::message
役割の指示文は **毎回の会話の冒頭で AI に渡される** ため、長すぎると会話の精度が落ちることがあります。最初は5〜10行程度から始めて、必要に応じて足していくのがおすすめです。
:::

:::message alert
1つの役割に「SNS集客もやって、契約書ドラフトもやって」と複数の専門性を詰め込むと、AI の応答が中途半端になりがちです。**役割は1つにつき1つの専門性** に絞ったほうが結果が安定します。
:::

:::details 役割をうまく書くコツ
1. 誰のために働くか（業種 / 規模 / 立場）
2. どんな成果物を出すか（投稿文 / リスト / 提案書）
3. トーンや形式の制約（文字数 / 丁寧体 or カジュアル / 表組み）
4. 避けるべきこと（断定 / 法的助言 / 個人情報の扱い）

この4点を順番に書くだけでも、応答の質が大きく変わります。
:::
