## この章でできるようになること

- 設定や運用で詰まったときに、確認すべき順序がわかる
- AI の応答が期待と違うときの調整方法がわかる
- 自動タスクが動かないときの切り分け手順がわかる

## OAuth 連携で詰まったときの確認順

外部サービス（Google・Notion・X など）との連携で「接続済みになっているはずなのに、AI が「カレンダーが見えません」と返す」「Notion のデータが取得できない」といった症状が出たとき、次の順で確認します。

1. **権限の許可範囲**：OAuth 画面で「許可」したスコープに、必要な権限が含まれているか
2. **アカウントの取り違え**：個人アカウントと業務アカウントを使い分けている場合、Teal に連携したのは想定どおりのアカウントか
3. **連携先サービスの状態**：Google / Notion / X の各サービスが正常稼働しているか
4. **トークンの期限切れ**：長期間使っていない連携は、サービス側でトークンが失効していることがあります。一度「切断」→「再接続」で更新

ここまでで解決しないときは、サイドバーの「外部連携」で該当サービスを切断し、最初から接続をやり直すのが確実です。

## AI の応答が思った内容と違うとき

「答えがふんわりしている」「同じ質問をしても毎回違う返事が来る」と感じたら、まず次の3点を見直します。

### 1. 役割の指示文に制約があるか

役割の指示文に **具体的な制約** が書かれていないと、AI は毎回違う書き方になりがちです。

```text
✗ あなたは SNS の専門家です。
○ あなたは BtoB 製造業向けの X 投稿を専門にする
   コピーライターです。投稿は120〜140字、
   ハッシュタグは最大3個、丁寧体で書いてください。
```

文字数・トーン・ハッシュタグ数のような **数えられる制約** が入ると、応答が安定します。

### 2. プロフィールが空欄のままになっていないか

サイドバーの「プロフィール」に会社名・業種・役職などが入っていないと、AI は毎回ゼロから前提を推測します。

「業種：BtoB 製造業（精密部品）」「役職：マーケ責任者」程度の情報を入れておくだけで、AI の応答の **抽象度が一段下がります**。

### 3. フィードバックを具体的に返しているか

「もう少しいい感じで」のようなフィードバックは、AI が学習しにくい指示です。

- ✗ 「微妙、書き直して」
- ○ 「冒頭の絵文字を外し、文末を疑問形にしてください」

第07章「スキル」でも触れましたが、**何を・どう変えるか** を明示するほど、次回以降の応答に反映されやすくなります。

## 自動タスクが動かないとき

自動タスクが想定の時刻に実行されない、または通知が届かないときは、次の順で確認します。

1. **役割が有効になっているか**：紐づいている役割が削除されていないか
2. **スケジュール設定**：曜日・時刻・タイムゾーンが意図どおりか
3. **通知先が空になっていないか**：「通知設定」で受信先が1つも選ばれていないと、実行しても通知が届きません
4. **連携先トークンの期限切れ**：自動タスクが Google や Notion に書き込む場合、連携トークンが期限切れだと実行が止まります
5. **会話履歴の確認**：Web ダッシュボードの会話履歴に「自動タスク実行ログ」が追加されているかをチェック

実行ログがあるのに通知だけ届かない場合は、3〜4 の通知設定・トークン状態の問題である可能性が高いです。

## アカウントが分かれてしまったとき

LINE で Teal を使い始めて、あとから Web でもサインアップしてしまうと、別ユーザーとして登録されることがあります。会話履歴が片方から見えないと感じたら、サイドバーの「外部連携」→「LINE」が「連携済み」になっているかを確認してください。連携済みになっていれば、Web と LINE は同じアカウントとして扱われます。

連携できていない状態で会話を続けてしまった場合は、現在使っているアカウントに LINE を後から連携するか、新しい方のアカウントを破棄して元のアカウントに統一するかを選ぶことになります。残したい会話履歴がある側を残してください。

## 「やってはいけない」の整理

ガイドや記事を社内で展開するときに、文章として **避けたい表現** をまとめておきます。Teal の説明として読み手に誤解を与えないためです。

- 「何でもできる」のような過度な表現
  - 具体的な機能名・ユースケースで語ったほうが、信頼性が伝わります
- 「人間が不要になる」といった誤解を招く表現
  - Teal は「AI が下書きを作り、人間が判断する」設計です
- 「LINE と Web だけ」のような連携先を少なく見せる記載
  - 標準で Google・LINE・Slack・Notion・X の5つに対応しています
