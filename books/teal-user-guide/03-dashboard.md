## ダッシュボードの全体構成

Web ダッシュボードは大きく3つのエリアに分かれています。**サイドバー（左）・メインエリア（中央）・チャット画面**です。サイドバーで「どこに移動するか」を決め、メインエリアまたはチャット画面で「何をするか」を進めます。

freee や Stripe のダッシュボードと似た発想で、左側にナビゲーション、右側に作業領域、という配置です。

## 操作手順

### Step 1. サイドバー（左ペイン）を確認する

サイドバーには次の項目が並びます。

1. **新しい会話** — 新規の会話（ルーム）を作成するボタン。押すと「ウィザード」が起動して、目的を選ぶだけで最適な会話が始まります
2. **会話履歴** — 過去の会話がリスト表示されます。クリックすればその場で再開できます
3. **アシスタント設定** — AI アシスタントの名前・一人称・応答スタイルなどパーソナリティを設定します
4. **プロフィール** — ユーザー自身の情報（会社名・業種・役職など）を登録します
5. **外部連携** — Googleカレンダー・Gmail・Notion・X（旧Twitter）など各サービスの接続設定です
6. **スキル** — AI が学習した能力の一覧を確認できます
7. **通知設定** — Web ダッシュボード / LINE / Slack のどこに通知を送るかを決めます

「アシスタント設定」と「プロフィール」は名前が似ていますが、役割が違います。**アシスタント設定は AI 側の人格、プロフィールはユーザー側の属性**、と覚えるとわかりやすいです。

### Step 2. メインエリア（中央ペイン）を見る

メインエリアはカード型の UI です。次の3つが中心になります。

1. **タスク提案カード** — 「今週の競合分析を行う」「ブログ記事を書く」など、すぐ実行できるタスクの候補がカード形式で並びます
2. **カテゴリタブ** — 業種・機能別のカテゴリがタブで切り替えられます。タブを選ぶと、そのカテゴリのサブカテゴリ（テンプレート集）が表示されます
3. **ウィザード** — 「新しい会話」ボタンを押すと起動する案内画面です。やりたいことを選んでいくと、最適な会話の出発点まで案内してくれます

タスク提案カードを **タップするだけ** で、そのタスクに合わせた質問とともに会話が始まります。「最初に何を聞けばいいか」をユーザーが考えなくていい点がカード型 UI の利点です。

### Step 3. チャット画面で会話する

会話を始めると、チャット形式の画面に切り替わります。

1. **テキスト入力** — キーボードで指示を入力します
2. **音声入力** — マイクアイコンをタップすると音声で指示できます
3. **ファイル添付** — 画像やドキュメントをアップロードして AI に読み取らせられます
4. **AI の応答** — リアルタイムでストリーミング表示されます

チャット画面の右上には、現在の会話と紐づいた「役割」を切り替えるメニューや、会話のタイトルを編集するメニューが配置されています。

## 「新しい会話」を始める2通りの方法

ダッシュボードからは、2通りの始め方があります。

| 方法 | 手順 | 向いている場面 |
| --- | --- | --- |
| A. ウィザード | サイドバーの「新しい会話」→ 目的を選ぶ → 推奨カテゴリへ案内される | やりたいことが言葉になっていない / 自分に合う使い方を探したい |
| B. カテゴリから直接 | メインエリアでカテゴリタブを選ぶ → サブカテゴリのカードをタップ | やりたいことが決まっている（SNS集客・契約書ドラフトなど） |

最初のうちは A でかまいませんが、慣れてくると B のほうが速いことが多いです。

## つまずきやすいポイント

:::message
会話履歴は古いものから順に並ぶわけではなく、**最後にメッセージを送った日時の新しい順** に並びます。久しぶりに使う会話を探すときは、サイドバーの検索欄にキーワードを入力したほうが早いです。
:::

:::details サイドバーが表示されないとき
画面幅が狭い場合（スマートフォンや小さなウィンドウ）、サイドバーは折りたたまれます。左上のハンバーガーアイコン（横線が3本のマーク）をタップすると展開できます。
:::
